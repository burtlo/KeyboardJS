Introduction

What is KeyboardJS?

KeyboardJS allows developers to leverage a powerful API to bind any key, or key combo. Its flexible and easy to use. KeyboardJS has no dependencies, and can be used as both a standard JavaScript library, or as an AMD module. See RequireJS for details.

Features

What features does KeyboardJS offer?

  • A powerful API thats easy to use, yet designed for very granular control.
  • Offers super flexible binding. All of the keys on the keyboard are treated equally. No special rules for modifier keys.
  • A detailed US locale. Bind to virtually any key on a US keyboard. Even symbols such as '$', '!', '?'.
  • Highly extensible. Add a custom locale for non US keyboards, or even custom key names. If you're making a game you could have a locale to map keys to 'jump', 'fire', etc...
  • Key Macros allow turning combos into keys. This is how KeyboardJS's US locale turns the combo 'shift + 1' into '!'
  • Developer support. If you need help or find a bug, I'll (Robert Hurst) be here to give you a hand. Just post an issue in the issue tracker. I'll continue to maintain KeyboardJS and assist those trying to use it.

Language Support

KeyboardJS can support any locale, however out of the box it just comes with the US locale (for now). Adding a new locale is easy. Map your keyboard to a locale object and pass it to KeyboardJS.locale.register(localeName, localeDefinition) then call KeyboardJS.locale(localeName).

If you create a new locale please consider sending me a pull request or submit it to the issue tracker so I can add it to the library.
